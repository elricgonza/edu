0 Preparar el Proyecto

Antes de hacer deploy, tu proyecto necesita estos archivos clave.

📁
Estructura de archivos requerida
requirements.txt · Procfile · .env · wsgi.py
▼

Tu repositorio debe tener esta estructura antes de hacer deploy:

estructura del proyecto
# Estructura mínima recomendada
mi-app/
├── app/
│   ├── __init__.py
│   ├── routes.py
│   ├── models.py
│   └── templates/
│       └── index.html
├── static/
│   └── css/
├── requirements.txt       # dependencias
├── Procfile               # comando de inicio
├── wsgi.py                # punto de entrada
├── .env.example           # vars de entorno (sin datos reales)
├── .gitignore             # excluir .env y __pycache__
└── runtime.txt            # versión de Python
requirements.txt

wsgi.py
from app import create_app

app = create_app()

if __name__ == "__main__":
    app.run()

.gitignore

1
Crear cuenta y conectar GitHub
render.com → Sign up with GitHub
▼

Ve a render.com, regístrate con tu cuenta de GitHub. Autoriza el acceso al repositorio de tu app.

💡 Tu código debe estar en un repositorio de GitHub (público o privado). Si no lo tienes, crea uno: git init → git push
Subir proyecto a GitHub
# Inicializar repositorio
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/TU_USUARIO/TU_REPO.git
git push -u origin main
2
Crear la base de datos PostgreSQL
New → PostgreSQL → Free tier
▼

En el Dashboard de Render: New + → PostgreSQL

  • Name: mi-app-db
  • Region: Ohio (US East) — el más rápido
  • Plan: Free
  • Clic en Create Database
  • Copiar el valor de "Internal Database URL"
⚠️ Guarda el Internal Database URL — lo necesitarás en el siguiente paso. Tiene el formato: postgresql://user:pass@host/dbname
3
Crear el Web Service
New → Web Service → conectar repositorio
▼

En Render: New + → Web Service, selecciona tu repositorio de GitHub.

Configuración del Web Service
Name:            mi-app
Region:          Ohio (US East)
Branch:          main
Runtime:         Python 3
Build Command:   pip install -r requirements.txt
Start Command:   gunicorn wsgi:app
Plan:            Free

Luego agrega las Environment Variables:

Variables de entorno
DATABASE_URL=postgresql://user:pass@host/dbname  # del paso anterior
SECRET_KEY=una-clave-secreta-muy-larga-y-aleatoria
FLASK_ENV=production
PYTHON_VERSION=3.12.3
💡 En tu app/__init__.py usa: app.config['SQLALCHEMY_DATABASE_URI'] = os.environ.get('DATABASE_URL')
4
Inicializar la base de datos
Migrations con Flask-Migrate o db.create_all()
▼

Una vez que el deploy esté corriendo, ejecuta las migraciones desde la consola de Render (Shell tab):

Opción A: Flask-Migrate (recomendado)
flask db init       # solo la primera vez
flask db migrate -m "initial"
flask db upgrade
Opción B: db.create_all() en Python
from app import create_app, db
app = create_app()
with app.app_context():
    db.create_all()
    print("✓ Tablas creadas")
💡 Puedes también agregar un Build Command con: pip install -r requirements.txt && flask db upgrade para que las migraciones corran automáticamente en cada deploy.
5
Tailwind CSS en producción
CDN Play vs Build process
▼

Para Tailwind en producción tienes dos opciones:

Opción A: CDN (más simple, para pruebas)
<!-- En tu base.html -->
<script src="https://cdn.tailwindcss.com"></script>
Opción B: Build con npm (producción real)
# instalar tailwind
npm install -D tailwindcss
npx tailwindcss init

# En tailwind.config.js:
content: ["./app/templates/**/*.html"]

# Build del CSS
npx tailwindcss -i ./static/css/input.css -o ./static/css/output.css

# En Render, Build Command:
npm install && npx tailwindcss build && pip install -r requirements.txt
6
Deploy continuo (CI/CD)
Push a GitHub = deploy automático
▼

Render detecta automáticamente cada push a tu branch main y redeploya. Para hacer cambios:

Workflow de desarrollo
# Editar código localmente
git add .
git commit -m "feat: nueva funcionalidad"
git push origin main
# → Render detecta el push y despliega automáticamente ✓
✅ Tu app estará disponible en: https://mi-app.onrender.com
🚂 Railway — $5 USD de crédito gratis/mes (aproximadamente 500 horas de ejecución). PostgreSQL incluido. Interfaz muy intuitiva.
1
Crear proyecto en Railway
railway.app → New Project → Deploy from GitHub
▼

Entra a railway.app, inicia sesión con GitHub, y crea un nuevo proyecto desde tu repositorio.

O usando Railway CLI
# Instalar CLI
npm install -g @railway/cli

# Login
railway login

# Inicializar en tu proyecto
railway init

# Deploy
railway up
2
Agregar PostgreSQL
New → Database → Add PostgreSQL
▼

Dentro de tu proyecto en Railway: + New → Database → Add PostgreSQL. Railway automáticamente expone la variable DATABASE_URL a tu servicio.

Variables disponibles automáticamente
DATABASE_URL          # URL completa (auto)
PGHOST                # host (auto)
PGPORT                # puerto (auto)
PGUSER                # usuario (auto)
PGPASSWORD            # contraseña (auto)
PGDATABASE            # nombre db (auto)

Solo agrega tus variables personalizadas:

Variables a agregar manualmente
SECRET_KEY=tu-clave-secreta
FLASK_ENV=production
3
Configurar el Start Command
Settings → Start Command
▼
Start Command en Railway Settings
gunicorn wsgi:app --bind 0.0.0.0:$PORT
💡 Railway inyecta la variable $PORT automáticamente. Gunicorn debe escuchar en ese puerto.
🪁 Fly.io — Tier gratuito incluye 3 VMs pequeñas, 3GB de almacenamiento y PostgreSQL básico. Usa Docker internamente, más control y rendimiento.
1
Instalar flyctl y login
CLI de Fly.io
▼
Instalar flyctl
# macOS / Linux
curl -L https://fly.io/install.sh | sh

# Windows (PowerShell)
iwr https://fly.io/install.ps1 -useb | iex

# Login

2
Lanzar la aplicación
fly launch — genera fly.toml automáticamente
▼
Desde la raíz de tu proyecto
fly launch
# → Detecta Python automáticamente
# → Crea fly.toml, Dockerfile, .dockerignore
# → Pregunta nombre, región, PostgreSQL
fly.toml generado (ajusta si es necesario)
[build]

[env]
  FLASK_ENV = "production"

[[services]]
  http_checks = []
  internal_port = 8080
  protocol = "tcp"

  [[services.ports]]
    handlers = ["http"]
    port = 80

  [[services.ports]]
    handlers = ["tls", "http"]
    port = 443
3
Crear PostgreSQL y conectar
fly postgres create + attach
▼
Crear y adjuntar base de datos
# Crear cluster de PostgreSQL
fly postgres create --name mi-app-db

# Adjuntar al app (expone DATABASE_URL automáticamente)
fly postgres attach mi-app-db

# Agregar SECRET_KEY
fly secrets set SECRET_KEY="tu-clave-secreta"

# Deploy
fly deploy
✅ Tu app estará en: https://mi-app.fly.dev
💜 Heroku — Ya no tiene tier gratuito. El plan Eco cuesta $5/mes (incluye dynos que duermen). PostgreSQL gratuito con límite de 10,000 filas (plan Essential $5/mes para producción real).
1
Instalar Heroku CLI y crear app
heroku.com → CLI → heroku create
▼
Setup inicial
# Instalar CLI desde heroku.com/cli
heroku login

# Crear app
heroku create mi-app-nombre

# Agregar PostgreSQL
heroku addons:create heroku-postgresql:essential-0

# Configurar vars de entorno
heroku config:set SECRET_KEY="tu-clave-secreta"
heroku config:set FLASK_ENV=production

# Deploy via git
git push heroku main

# Ejecutar migraciones
heroku run flask db upgrade
✅ Tu app estará en: https://mi-app-nombre.herokuapp.com
